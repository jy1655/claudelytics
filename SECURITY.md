# Security Policy

## Supported Versions

Security fixes are provided for the latest state of the `main` branch in this fork.

## Reporting a Vulnerability

Please report vulnerabilities privately first.

For now, open a GitHub issue with minimal public detail and include:

- affected command/path
- impact
- reproduction steps
- suggested mitigation (if available)

If the issue contains sensitive details, request a private contact channel in the issue.

## Response Expectations

- Initial triage target: within 7 days
- Status updates: provided in the tracking issue
- Fix release: as soon as a safe patch is validated
